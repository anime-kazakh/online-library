# Подключение статических файлов.

Static файлы - это например CSS, JS, изображения и др.

Есть стандартные пути это когда директория static располагается в папке приложения.
Это все которые объявлены в `INSTALLED_APPS`.

Не стандартные пути это когда директория располагается в других частях проекта.
Их нужно объявлять в переменной `DIR`.

А так же static располагается в корневой папке проекта.

Когда `DEBUG=True` тестовый веб сервер подтягивает статические файлы из стандартных путей.
Но перед продом их нужно переместить в корневую папку проекта.

Для этого есть команда, которая автоматический собирает статические файлы:

```commandline
python manage.py collectstatic
```

Для подключения статических файлов необходимо определить переменные:
* `STATIC_URL` - префикс URL-адреса для статических файлов.
* `STATIC_ROOOT` - петь к корневой статической папке, формируется автоматически при запуске `collectstatic`.
* `STATICFILES_DIRS` - список дополнительных нестандартных путей к статическим файлам, используемых для сбора
и для режима отладки.

Подключение статических файлов происходит вот так:
```HTML
{% load static %}

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Моя библиотека</title>
    <link type="text/css" href="{% static 'home/css/style.css' %}" rel="stylesheet">
```
